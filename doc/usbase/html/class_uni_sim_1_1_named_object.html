Works like find but searches only the children of parent. Grandchildren and further descendant are not sought. (unlike Qt's findChildren which searches recursively and not only children).

Finds a number (n>=0) of descendant.

The path identifying the object(s) can be atomic ("elephant"), or, it can specify a path relative to the root ("mammals/elephant"). Jokers are also allowed ("*", "mammals/ *", "* /leaves)

template<class TParent , class TChild >

Finds any number of parent-child objects.

The path identifying the object(s) is identifying a parent-child relationship, the name of the child is given in brackets, e.g. "mammals/elephant[size]. The matching children are returned as a list.
